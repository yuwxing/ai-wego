## 执行总结

### 项目目标
将清华大学开源的 **OpenMAIC**（多智能体互动课堂平台）集成到 ai-wego 平台，打造"求职课堂"场景，提供 AI 驱动的沉浸式求职培训体验。

### 核心价值
| 维度 | 价值 |
|------|------|
| **产品差异化** | 业界首个多智能体求职培训平台，AI 面试官 + AI 导师 + AI 同学 |
| **用户体验** | 沉浸式模拟面试、实时反馈、个性化指导 |
| **技术优势** | 复用成熟开源框架，降低开发成本 70%+ |

### 推荐方案
**方案 A（快速上线）**：OpenMAIC 部署到 Vercel，通过 iframe 嵌入 ai-wego

### 多智能体协作机制

OpenMAIC 采用 **Director-Agent** 架构：

```
┌─────────────────────────────────────────────────────────┐
│                      Director (导演)                     │
│  - 决策哪个智能体发言                                     │
│  - 管理对话流程                                          │
│  - 使用 LangGraph 实现状态机                              │
└─────────────────────────────────────────────────────────┘
                            │
        ┌───────────────────┼───────────────────┐
        ▼                   ▼                   ▼
┌───────────────┐   ┌───────────────┐   ┌───────────────┐
│  Agent 1      │   │  Agent 2      │   │  Agent N      │
│  (AI 老师)    │   │  (AI 同学)    │   │  (AI 助教)    │
└───────────────┘   └───────────────┘   └───────────────┘
```

## 部署方案

### 方案对比

| 方案 | 优点 | 缺点 | 推荐度 |
|------|------|------|--------|
| **A: Vercel 部署 + iframe** | 快速上线、零维护、自动扩展 | 功能受限、样式隔离 | ⭐⭐⭐⭐⭐ |
| **B: 子路径部署** | URL 统一、品牌一致 | 部署复杂、资源共享难 | ⭐⭐⭐ |
| **C: Docker 自托管** | 完全可控、定制灵活 | 运维成本高、需要服务器 | ⭐⭐ |

### 方案 A 详解：Vercel + iframe

#### 部署步骤

```bash
# 1. Fork OpenMAIC 到自己的 GitHub 仓库
# https://github.com/THU-MAIC/OpenMAIC

# 2. 一键部署到 Vercel
# 访问: https://vercel.com/new/clone?repository-url=https://github.com/YOUR_NAME/OpenMAIC

# 3. 配置环境变量
# 在 Vercel Dashboard -> Settings -> Environment Variables 中添加:
# - DEEPSEEK_API_KEY (推荐，便宜)
# - DEFAULT_MODEL=deepseek:deepseek-v4-pro
# - ACCESS_CODE (可选，保护访问)

# 4. 获取部署 URL
# 例如: https://openmaic.vercel.app
```

#### 环境变量配置（推荐 DeepSeek）

```env
# LLM 配置（DeepSeek 性价比最高）
DEEPSEEK_API_KEY=sk-xxxxxxxxxxxx
DEEPSEEK_BASE_URL=https://api.deepseek.com
DEFAULT_MODEL=deepseek:deepseek-v4-pro

# TTS 配置（可选，面试场景需要语音）
TTS_DEEPSEEK_API_KEY=sk-xxxxxxxxxxxx
# 或使用免费的 MiniMax
TTS_MINIMAX_API_KEY=xxxxxxxx
TTS_MINIMAX_BASE_URL=https://api.minimaxi.com

# 访问控制（可选）
ACCESS_CODE=your-secret-code
```

#### iframe 嵌入方式

```html
<!-- 基础嵌入 -->
<iframe 
  src="https://openmaic.vercel.app"
  width="100%"
  height="800px"
  frameborder="0"
/>

<!-- 带参数嵌入 -->
<iframe 
  src="https://openmaic.vercel.app?topic=前端面试&agents=面试官,导师"
  width="100%"
  height="900px"
  frameborder="0"
/>
```

#### URL 参数说明

| 参数 | 说明 | 示例 |
|------|------|------|
| `topic` | 课堂主题 | `topic=前端工程师面试` |
| `agents` | 启用的智能体 | `agents=面试官,导师` |
| `lang` | 语言 | `lang=zh` |
| `embed` | 嵌入模式（隐藏导航） | `embed=true` |

```
│   用户行为   │────▶│  ai-wego 前端 │────▶│ OpenMAIC     │

                     │  - 用户简历/岗位信息传递                 │
                     └──────────────────────────────────────────┘
```

### 通信协议（postMessage）

```typescript
// ai-wego -> OpenMAIC
interface WegoToOpenmaicMessage {
  type: 'SET_RESUME' | 'SET_POSITION' | 'START_INTERVIEW' | 'END_INTERVIEW';
  payload: {
    resumeText?: string;
    position?: string;
    difficulty?: 'easy' | 'medium' | 'hard';
  };
}

// OpenMAIC -> ai-wego
interface OpenmaicToWegoMessage {
  type: 'INTERVIEW_COMPLETE' | 'INTERVIEW_FEEDBACK' | 'RESUME_ANALYSIS';
  payload: {
    summary?: string;
    scores?: {
      technical?: number;
      communication?: number;
      problemSolving?: number;
    };
    suggestions?: string[];
  };
}
```
